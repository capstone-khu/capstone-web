# Capstone

캡스톤 디자인 서비스의 **프론트엔드 작업 공간.** 화면 기획(UI 프로토타이핑)에서 출발해, 실제 프론트엔드 구현을 진행한다.

## 작업 전 반드시 — 기획 문서 먼저 읽기

**어떤 작업이든 코드를 건드리기 전에 `docs/plan/` 의 기획 문서를 먼저 읽는다.** 화면 구성·흐름·인터랙션의 단일 기준(source of truth)이다.

- `docs/plan/IA.md` — 전체 화면 구조와 흐름
- `docs/plan/USER-SCENARIO.md` — 사용자 시나리오
- `docs/plan/requirements/*.md` — 화면별 상세 명세 (로그인 / 곡선택 / 협주선택 / 연주 / 결과 / 마이페이지)

작업이 특정 화면을 건드리면, 그 화면의 `docs/plan/requirements/<화면>.md` 를 반드시 확인하고 명세와 어긋나지 않게 구현한다. **명세와 코드가 충돌하거나 명세에 없는 변경이면 먼저 사용자에게 알린다.**
