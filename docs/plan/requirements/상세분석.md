# AI 상세 분석

> 연주가 끝난 뒤, 실시간 멀티 에이전트(음정/박자/자세 + 슈퍼바이저)가 남긴 **세션 기록을 LLM 코치가 읽어** 디브리핑 + 영역별 연습 코칭으로 정리해주는 화면. 연주 중 실시간 교정과 달리, 곡 전체를 돌아보는 '레슨 디브리핑'에 해당한다.
>
> 두 가지 진입 맥락: **방금 끝난 연주**(결과 화면에서 진입) / **과거 기록 열람**(마이페이지 → 결과 경유 진입). `결과` 화면의 'AI 상세 분석' 버튼으로 들어온다.

## ① 상단 헤더 (App Header)

- **구성**: 좌측 '<' 뒤로가기 + 타이틀('AI 상세 분석 · {곡명}' 또는 과거 기록이면 '{날짜} 연주 · AI 상세 분석'). 우측: '홈으로'/'목록으로'(아웃라인).
- **인터랙션**: '<' → 직전 결과 화면으로. '홈으로'(방금 연주) → 곡 선택(홈) / '목록으로'(과거 기록) → 마이페이지.
- **이동 화면**: < → 결과 | 홈으로 → 곡 선택(홈) | 목록으로 → 마이페이지

## ② 분석 로딩 (Loading)

- **기능**: 진입 직후 LLM 분석을 흉내 내는 짧은 로딩. **3-dot 인디케이터** + 'AI가 이번 연주를 분석하고 있어요…' 안내.
- **인터랙션**: 없음(자동). 잠시 후 본문(③~)으로 전환. (실제 연동 시 LLM 호출 완료 시점에 해제)

## ③ 디브리핑 카드 (Debrief)

- **구성**: 카드 상단 전체 폭 검정 띠(좌측 'AI 상세 분석') + 한 줄 헤드라인(예: "이번엔 음정이 제일 아쉬웠어요") + 코치 코멘트 한 단락.
- **내용**: 가장 자주 흔들린 영역을 짚고, 자세가 음정·박자와 함께 흔들린 경우 그 연관(교차 도메인 원인)을 한 문장으로 언급. **사람 선생님 말투(전문용어 없이)**.
- **인터랙션**: 없음.

## ④ 영역별 탭 (Area Tabs)

- **구성**: **음정 / 박자 / 자세 3개 탭**(아이콘 + 이름 세그먼트 컨트롤, 선택 탭은 영역 컬러). 선택한 탭 아래에 그 영역의 **수준 배지 + '어디서·뭐가·왜 문제였는지' 한 줄 + (보통·아쉬움이면) '이렇게 연습' 블록**(연습 이름 + 방법). 진단과 처방을 한 곳에 합침.
- **기본 선택**: 가장 아쉬운 영역이 처음부터 열림.
- **수준(LLM 분류)**: 각 영역을 **잘함 / 보통 / 아쉬움** 중 하나로 **LLM이 분류**. '잘함' 탭은 칭찬 코멘트만(연습 블록 없음). (MVP는 피드백 개수로 흉내 — 0회 잘함·1회 보통·2회+ 아쉬움)
- **사유 기반(지향):** 막힌 마디·실패 사유(state) 데이터를 바탕으로 "어느 부분이 왜 문제였는지"를 LLM이 구체적으로 코치. (MVP는 영역·수준별 템플릿)
- **반복 약점 마디**: 여러 레슨에 걸쳐 같은 마디가 반복 실패하면(세션 간 누적 3회) '마디 N 집중 반복 레슨'을 함께 노출(시작은 마이페이지/이 화면 CTA).
- **인터랙션**: 탭 전환으로 영역별 내용 보기.

## ⑤ 하단 액션 (Footer Actions)

- **구성**: '결과 다시 보기'(아웃라인, lg) + '홈으로'/'목록으로'(검정, lg) 2버튼.
- **인터랙션**: '결과 다시 보기' → 직전 결과 화면. '홈으로'/'목록으로' → 헤더 우측과 동일.
- **이동 화면**: 결과 다시 보기 → 결과 | 홈/목록 → 곡 선택(홈)/마이페이지

---

## 데이터 / 생성 메모 (구현 참고)

- **입력**: 그 연주의 세션 기록 — 영역별 피드백 요약, 막힌 마디·전환(SWITCH) 지점, 반복 연습 결과, 사용자 과거 맥락(누적 약점·Q테이블). LLM은 원본 오디오/영상을 재분석하지 않고 이 기록만 읽는다.
- **호출 시점**: 'AI 상세 분석'을 누른 시점에 1회 생성 → 저장(캐시). 같은 연주는 재호출하지 않고 저장된 리포트를 보여준다.
- **MVP**: 실제 LLM 대신 세션 요약(영역별 개수)에서 리포트를 결정적으로 생성하는 목으로 대체한다.
